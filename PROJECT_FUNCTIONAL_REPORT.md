## 1. 项目概述

**Couple Space** 是一款专为情侣设计的渐进式Web应用（PWA），旨在通过一系列实用、有趣且充满情感互动的功能，增强情侣间的沟通与连接，共同记录生活中的点滴美好。项目秉持**女性友好**的设计原则，在功能规划和视觉呈现上充分考虑女性用户的偏好和体验，致力于打造一个温暖、私密且富有设计感的数字二人世界。

经过多轮迭代，应用现已集成了超过 **20个** 核心功能模块，涵盖了从日常记录、任务协作到健康生活、情感互动的方方面面。项目采用现代化的技术栈，确保了高性能、高可靠性和优秀的用户体验。

---

## 2. 核心设计理念

### 2.1. 女性友好的功能设计

项目的核心驱动力之一是为女性用户提供贴心和愉悦的体验。这一理念贯穿于多个功能模块，例如：

- **经期追踪**：提供精准、便捷的生理周期记录与预测功能。
- **减肥健身**：不仅是数据记录，更加入了情侣互相点赞、评论鼓励的社交元素，将枯燥的健康管理变得充满温情。
- **点菜板**：解决了“今天吃什么”的日常难题，通过分区布局和随机点菜增添趣味性。
- **视觉美学**：采用网易云音乐风格的勋章、柔和的渐变色和细腻的动画效果，满足女性用户对视觉美学的追求。

### 2.2. 增强情侣连接

所有功能都围绕“**共同**”和“**互动**”两个关键词展开，旨在将个人行为转化为情侣的共同体验。

> 例如，“情侣挑战”功能允许双方设定共同目标并追踪各自的进度，通过评论区互相打气，完成挑战时的庆祝动画则为共同的努力赋予了满满的仪式感。这种设计将个人成长与情侣关系的发展紧密联系在一起。

### 2.3. 统一且富有设计感的UI/UX

项目在视觉上追求高度的统一性和设计感，主要体现在：

- **温暖渐变背景 (`gradient-warm-subtle`)**：全局使用由粉色、紫色和蓝色构成的柔和渐变背景，营造出温暖、浪漫的氛围。
- **玻璃态效果 (`glass`)**：卡片、导航栏等元素广泛采用毛玻璃效果，增加了界面的层次感和现代感。
- **一致的深色模式**：所有页面和组件都经过精心适配，确保在深色模式下依然保持美观和可读性。

## 3. 功能模块详解

项目功能可划分为五大核心模块，下表对主要功能进行了详细梳理。

### 模块一：生活记录与回忆

| 功能点 | 详细描述 |
| :--- | :--- |
| **相册** | 支持上传和查看情侣间的照片，提供搜索、筛选和批量删除功能。 |
| **日记** | 共同或独立撰写日记，记录心情和故事，同样支持搜索、筛选和批量操作。 |
| **心情记录** | 每日选择一个表情记录当天心情，形成心情日历。 |
| **大事记** | 以时间线的形式记录情侣关系中的重要里程碑事件。 |
| **时光胶囊** | 将文字或图片封存到“时光胶囊”中，设定在未来的某个特定日期开启。 |
| **足迹地图** | 在地图上标记两人一起去过的地方，生成专属的情侣足迹地图。 |

### 模块二：任务与目标管理

| 功能点 | 详细描述 |
| :--- | :--- |
| **任务清单** | 创建共享的待办事项，分工协作，共同完成生活中的任务。 |
| **100件事** | 共同完成一份“情侣必做的100件事”清单，通过环形进度图可视化进度。 |
| **情侣承诺** | 许下对彼此的承诺，并追踪承诺的完成进度。 |
| **情侣挑战** | 设定共同的挑战目标（如运动、减肥、习惯养成等），追踪双方进度，并通过评论区互相鼓励。 |
| **成就系统** | 将情侣的互动行为游戏化，通过完成特定任务解锁成就勋章（网易云音乐风格），提供持续的激励。 |

### 模块三：健康与生活方式

| 功能点 | 详细描述 |
| :--- | :--- |
| **经期追踪** | 专为女性用户设计，提供生理周期记录、预测和提醒功能。 |
| **减肥健身** | 记录体重和运动数据，设定减肥和运动目标，并加入了情侣互相点赞和评论的互动功能。 |
| **点菜板** | 解决“吃什么”的难题。双方可维护自己的菜单，并可随机组合生成一餐的搭配，支持分区（荤/素/主食等）。 |

### 模块四：情感互动与趣味

| 功能点 | 详细描述 |
| :--- | :--- |
| **纪念日** | 记录所有重要的纪念日，并在首页提供下一个纪念日的倒计时。 |
| **愿望清单** | 共同维护一个愿望清单，互相了解对方的心愿。 |
| **留言板** | 给对方留下甜蜜的言语或提醒。 |
| **倒计时** | 为期待的事件（如旅行、生日）创建倒计时。 |

### 模块五：财务管理

| 功能点 | 详细描述 |
| :--- | :--- |
| **共享账本** | 记录共同的开销，支持分类统计，并通过饼图清晰展示支出构成。 |

## 4. 技术架构与实现

项目采用前后端分离的现代化Web架构，确保了开发效率和应用性能。

- **前端 (Client)**: 使用 **Vite + React + TypeScript** 构建，响应式UI框架为 **TailwindCSS**，路由管理为 **Wouter**。
- **后端 (Server)**: 基于 **Node.js** 和 **Express**，使用 **tRPC** 提供类型安全的API接口。
- **数据库 (Database)**: 采用 **PostgreSQL**，ORM工具为 **Drizzle ORM**，实现了高效的数据库操作和迁移。
- **PWA 支持**: 通过 **Workbox** 实现了完整的渐进式Web应用功能，支持离线访问和添加到主屏幕。
- **UI组件**: 基于 **shadcn/ui** 组件库，保证了UI的一致性和可访问性。

## 5. 未来展望

基于当前稳固的功能基础，未来可从以下几个方向进行深化和扩展：

1.  **智能化与个性化**：
    *   引入AI分析，根据情侣的互动数据生成“亲密度报告”或“年度总结”。
    *   允许用户自定义首页布局和主题颜色。

2.  **社交连接扩展**：
    *   增加“情侣广场”，匿名分享有趣的情侣挑战或生活趣事。
    *   允许将成就或完成的挑战分享到外部社交媒体。

3.  **功能深化**：
    *   为“足迹地图”增加路线规划和游记功能。
    *   为“共享账本”增加预算管理和周期性账单功能。

4.  **多端体验**：
    *   虽然PWA已具备跨平台能力，但可考虑使用 **Tauri** 或 **Electron** 封装成桌面应用，或使用 **React Native** 开发原生移动应用，以提供更极致的性能和系统集成能力。
